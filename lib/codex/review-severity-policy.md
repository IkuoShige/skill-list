# レビュー深刻度分類ポリシー

## 深刻度定義

### P0: Critical（致命的）
修正必須。マージ不可。
- セキュリティ脆弱性（インジェクション、認証バイパス、秘密情報の露出等）
- データ損失・破壊のリスク
- 本番障害を引き起こすバグ（クラッシュ、無限ループ、リソースリーク等）
- 既存機能の破壊（回帰バグ）

### P1: Important（重要）
修正推奨。マージ前に対応すべき。
- ロジックエラー（エッジケース未処理、off-by-one等）
- パフォーマンス問題（O(n²)で回避可能、不要なDB/APIコール等）
- エラーハンドリングの欠如（例外が握り潰される等）
- API契約違反（型不一致、未定義の戻り値等）

### P2: Suggestion（提案）
任意対応。改善提案。
- 命名の改善
- コードの簡素化・可読性向上
- コメント追加の提案
- テストカバレッジの提案
- コーディング規約との軽微な不一致

## 表示ルール

1. **P0は冒頭にハイライト表示する**
   ```
   ## P0: Critical Issues
   - **[ファイル名:行番号]** 説明
   ```

2. P1はP0の後に表示する
   ```
   ## P1: Important Issues
   - **[ファイル名:行番号]** 説明
   ```

3. P2は最後に表示する
   ```
   ## P2: Suggestions
   - **[ファイル名:行番号]** 説明
   ```

4. 該当なしのカテゴリは省略する
5. P0がゼロの場合は「P0指摘なし」と明記する
6. 各指摘にはファイル名と行番号を必ず含める
7. 修正案がある場合はコードブロックで提示する

## サマリー
レビュー結果の末尾に以下のサマリーを付ける:
```
---
P0: N件 | P1: N件 | P2: N件
```

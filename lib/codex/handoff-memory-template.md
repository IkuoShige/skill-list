# 引き継ぎ文書テンプレート

以下のテンプレートに従い、引き継ぎ文書を生成する。

## テンプレート

```markdown
# セッション引き継ぎ: {YYYY-MM-DD HH:MM}

## 目標
- このセッションで達成しようとしていたこと

## 現状
- 完了したタスク
- 進行中のタスク（進捗率の目安）
- 未着手のタスク

## 決定事項
- セッション中に下した設計判断・方針決定（理由を含む）

## 未解決事項
- ブロッカー・障害（原因の仮説があれば記載）
- 判断保留中の事項

## 注意事項
- ハマりポイント・罠
- 回避策を取った箇所とその理由

## 関連ファイルパス
- 変更したファイル（git diffから取得）
- 参照が必要なファイル

## 再現コマンド
- ビルド・テスト・起動等の主要コマンド
- 問題再現のためのコマンド（該当する場合）

## Training State（Training実行中の場合）
- Training PID: {pid}
- Run ID: {HOLO_RUN_ID}
- Log file: {path}
- Last metrics: iter={N}, reward={R}, loss={L}
- Status: running / completed / failed
```

## 記入ガイドライン
- 各セクションが空の場合は「なし」と記載する（セクション自体は省略しない）
- 決定事項には必ず「なぜ」を含める
- ファイルパスは絶対パスで記載する
- コマンドはコピペで即実行できる形式にする

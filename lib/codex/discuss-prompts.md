# 議論用 固定プロンプトテンプレート

サブエージェントがこのファイルを読み、指定ステップのテンプレートに変数を埋めてCodexに送信する。

## Step 1: 問い定義

```
あなたはClaude Code（CC）との構造化議論のファシリテーターです。

テーマ: {theme}

思考原則:
- 相手の主張に同意する前に、まずその主張の最も弱い点を特定せよ
- 弱点が見つからないなら、自分の理解が浅い可能性を疑え
- 迎合は合意ではない。早すぎる収束は思考の放棄である

タスク: 上記テーマについて、問いの本質を1文で定義せよ。立場はまだ出さないこと。
```

## Step 3: Codex独自立場生成

```
先ほど定義した問いの本質に対して、あなた自身の独立した初期立場を生成してください。
検証・証明の根拠として何が十分で何が不十分かを主張してください。
立場は具体的に。曖昧な「バランスが大事」系の回答は不可。
```

## Step 4: 批評・論点整理

```
以下の2つの独立した立場を批評し、論点を整理してください。

## CCの立場
{cc_position}

## Codexの立場
{codex_position}

---
両立場の強い点・弱い点を指摘し、対立軸を整理してください。合意に向けた論点を提示してください。
```

## Rally: 相違点解消（ラリー継続用）

```
CCの応答は以下です。相違点を解消し、合意に向けて指揮してください。

{cc_response}

---
相違点について、CCの反論の弱い点を指摘した上で、合意案を提示してください。無理な妥協は不要です。
```

## Step 8: 合意構造化

```
完全合意に達しました。合意内容を以下の構造で整理してください:
1. 合意の前提（何を検証する議論だったか）
2. 合意事項（確定版）
3. 先送り事項
4. 合意に至った論理の流れ（要約）
```
